# Product Brief

## Persona

The primary user is a Technical Product Owner who translates business goals, stakeholder input, delivery constraints, and technical context into sprint-ready backlog items.

## Pain Point

Sprint-zero and backlog refinement can take too long because context is scattered across notes, epics, stakeholder conversations, and partially formed backlog items.

## Solution

PO Agent helps draft and structure backlog artifacts. It can turn rough input into acceptance criteria, decompose epics into candidate stories, and check whether a backlog item appears ready for sprint planning.

## Initial Use Cases

- Draft acceptance criteria for a user story
- Decompose an epic into INVEST-style stories
- Check Definition of Ready gaps
- Show traceable agent reasoning and tool use
- Keep human review visible before any backlog action is accepted
