# LLM Adapter

## Provider Abstraction

The LLM layer exposes a common provider method:

```python
generate_structured_output(
    task: str,
    system_prompt: str,
    user_input: str,
    context: dict,
    expected_schema: dict,
) -> dict
```

This keeps the PO Agent independent from any single LLM vendor or SDK.

## Mock Provider

`MockLLMProvider` is the default. It requires no API key and returns the deterministic structured output generated by the existing rule-based agent. This keeps the project runnable for local demos and interviews.

## OpenAI-Compatible Provider

`OpenAICompatibleProvider` is optional. It reads configuration from environment variables:

- `LLM_API_KEY`
- `LLM_BASE_URL`
- `LLM_MODEL`
- `LLM_TIMEOUT_SECONDS`

If configuration is missing, the provider is not used and the agent falls back to mock mode.

## Runtime Modes

`mock` mode is deterministic and local.

`llm` mode attempts provider generation. It is safe to request even without API keys because the provider factory and JSON guard fall back to mock output.

## JSON Guard

The JSON guard prevents backend crashes and unsafe provider output by:

- safely parsing JSON
- checking required top-level keys
- falling back to mock output when parsing or validation fails

## Fallback Strategy

Every agent run starts with deterministic output. LLM output is treated as an optional enhancement, not a required dependency. If provider configuration, network calls, JSON parsing, schema validation, or model validation fails, the deterministic output is returned.

Runtime metadata records:

- requested mode
- used mode
- provider
- fallback usage
- fallback reason

## Avoiding Vendor Lock-In

The provider abstraction can support OpenAI-compatible APIs, Claude Agent SDK, enterprise gateways, or local models later. The rest of the agent does not need to know which provider produced the structured output.

## Future Claude Agent SDK Integration

A Claude Agent SDK adapter could implement the same provider method and use the existing MCP-style tool context, prompt templates, JSON guard, and runtime metadata pattern.
